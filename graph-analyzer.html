❓ Graph Analyzer — Help

Input Format

Paste one directed edge per line as two node names separated by a space or tab:

WTP       V-E1-300
V-E1-300  V-CDFG-500
V-E1-300  V-AB-350
WTP       WTP

Each line becomes a directed edge from the left node to the right node. Lines with more or fewer than two tokens are flagged as Malformed. A node pointing to itself is flagged as a Self-Loop and skipped from the graph.

Preprocessing Modes

  • Do Not Alter Input Names — node names used exactly as typed.
  • Ignore Symbols Only — strips non-alphanumeric characters, uppercases. Useful when the same node appears as V-E1 and VE1.
  • Ignore Prefix & Symbols — additionally strips any leading letters, leaving only the numeric part. Useful when node numbers are significant but text prefixes vary.

Analysis Panels

  • Repeated / Bidirectional — A→B appears more than once (Repeated), or both A→B and B→A exist (Bidirectional).
  • Triangles — three mutually connected nodes: A–B, B–C, A–C all present.
  • Self-Loops — a node listed as its own neighbour (A→A).
  • Nodes with 4+ Branches — nodes with four or more connections; highlighted red in the graph.
  • Malformed Lines — lines that did not parse as exactly two tokens.

Click any highlighted item in the sidebar to zoom the graph to that node or group.

Export Files

The two JSON exports correspond to the two files each zone needs in this project:

📁 zone.json  (repo — static)

{
  "nodes": [
    {
      "data": {
        "id": "WTP",
        "lat": null,
        "lng": null
      },
      "position": { "x": 320, "y": 240 }
    }
  ],
  "edges": [
    {
      "data": {
        "id": "WTP_V-E1",
        "source": "WTP",
        "target": "V-E1"
      }
    }
  ]
}

Topology only. Add real lat/lng values to each node after export to enable GPS layout in the SCADA visualizer.

☁️ zone_status.json  (gist — live)

[
  {
    "id": "WTP",
    "type": "pump",
    "label": "WTP\nPump Stn",
    "state": "OFF"
  },
  {
    "id": "WTP_V-E1",
    "flow": ""
  }
]

Flat array consumed by applyStatus(). Edit type, label, and state per node after export. Valid types: pump, valve, zone.
